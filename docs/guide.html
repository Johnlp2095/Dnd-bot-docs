🎯 Pro Search Tips

✅ Good Search Strategies

  • Use Ctrl+F for browser search
  • Search for method names like "make_attack"
  • Search for component names
  • Try keywords like "damage" or "initiative"
  • Search for parameter names

❌ Ineffective Searches

  • Very generic terms like "the" or "a"
  • Typos in method names
  • Using spaces in method names (use underscores)
  • Case-sensitive searches (search is case-insensitive)
  • Partial words without context

❓ Frequently Asked Questions

How do I find a specific method?

Use the Search Index page and press Ctrl+F to search, or type in the search box at the top to filter results. Methods are marked with a green "Method" badge for easy identification.

Where can I see code examples?

Every method on the component pages includes an "Example" section with working Python code. The examples are formatted with syntax highlighting and ready to copy-paste into your project.

What if I need to use multiple components?

Each component page includes a "Related Components" section showing how it connects to other systems. Follow the cross-reference links to understand the full workflow. For example, Combat System uses Dice Roller and Character System.

Is there a keyboard shortcut for searching?

Yes! Use Ctrl+F (or Cmd+F on Mac) to open your browser's built-in search. This is the fastest way to find keywords on any page. The search index page also has a filter box for additional filtering options.

What does the cross-reference section mean?

Cross-references show which other methods or components are related to the current one. For example, "make_attack()" method links to both "Dice Roller" (for rolling attacks) and "Character System" (for the target character). Click these links to learn more.

Can I use this documentation offline?

Yes! Since this is hosted on GitHub Pages, you can clone the repository and open the HTML files directly in your browser. All styles and functionality work without an internet connection.

What if I find an error in the documentation?

This is a student project! If you find errors, you can submit an issue on GitHub. Documentation is maintained as part of the toolkit's lifecycle and improvements are always welcome.

🏆 Best Practices

Read Complete Method Signatures

Always check the full method signature (like the parameters and return type) before using a method. This is usually shown in a gray code box.

Follow Code Examples

Every method has working examples. These show you the exact syntax and expected results. Use them as a template for your code.

Check Related Components

When learning a method, check its related components to understand how it fits into the bigger picture. This prevents integration issues later.

Use the Search Index

The search index is your fastest route to finding specific keywords, parameters, or methods. Bookmark it or keep it in a tab.
